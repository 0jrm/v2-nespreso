# Engineering Principles Agent Pack

A token-efficient, platform-agnostic bundle of engineering skills and lifecycle hooks. The skill files are assembled from verbatim spans of `src/source.txt` using `extract.py`; the hooks, index, and readme are compact operational wrappers generated by `build.sh`.

```text
eng-principles-pack/
├── skills/
│   ├── design-review.md

```

The archive root also includes:

```text
build.sh
extract.py
src/source.txt
```

Run `./build.sh` from the archive root to regenerate the package from source.

## Usage model

Start with `INDEX.md`. Select one lifecycle hook when relevant, then load the smallest set of skill files needed for the current task. This keeps context narrow while preserving source-grounded guidance.

Good default patterns:

- Code review: `hooks/pre-pr-checklist.md` + `skills/design-review.md` + `skills/testing-strategy.md`.
- Bug fix: `hooks/post-bugfix.md` + `skills/debugging.md` + `skills/testing-strategy.md`.
- API change: `skills/api-design.md` + `skills/error-handling.md`.
- Numerical/scientific PR: `hooks/pre-merge-numerical.md` + `skills/numerical-code.md` + `skills/ai-code-verification.md` if AI helped write the code.

## Platform wiring

### Claude-style skills

Create one skill directory per file and copy the Markdown content into that skill's `SKILL.md`. Use the skill filename as the trigger-oriented skill name. Keep `INDEX.md` as a top-level routing document.

Example:

```text
.claude/skills/design-review/SKILL.md
.claude/skills/error-handling/SKILL.md
```

### Cursor-style rules

Copy each skill or hook into a rule file, preserving the same names. Use always-on rules sparingly; most files should be manually or semantically triggered.

Example:

```text
.cursor/rules/design-review.mdc
.cursor/rules/pre-pr-checklist.mdc
```

### Codex / AGENTS.md / generic agents

Reference `INDEX.md` from your root agent instructions and tell the agent to load only the matching files. Hooks can be pasted into automation steps, CI checklist comments, or pre-PR templates.

Example instruction:

```text
For engineering guidance, read eng-principles-pack/INDEX.md first. Load only the hook and skill files whose triggers match the current task.
```

## Extraction workflow

`extract.py` supports `bof`/`eof` sentinels and exact line-prefix markers. Multiple `--start`/`--end` pairs may be chained to append several source spans into one output file.

Example:

```bash
python3 extract.py src/source.txt out.md \
  --start bof --end eof \
  --start "Narrow Interfaces" --end "Idempotency and Safety"
```

This allows the pack to stay editable while keeping the skill body text traceable to the original source document.
